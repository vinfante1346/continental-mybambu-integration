Continental Assist API Integration Flow

Complete 8-Step Purchase & Query Workflow

1

Get Categories

consulta_categorias

Retrieve all plan categories assigned to your agency account. Categories determine the types of travel assistance plans available.

Purpose
Initial step to discover available plan categories
Returns
List of category IDs and names
2

Get Origins

consulta_origenes
consulta_origenes_x_iso

Query available origin countries where travelers will depart from. Use ISO-based endpoint for country code lookups.

User Action
Customer selects departure country
Returns
Available origin countries with codes
3

Get Destinations

consulta_destinos

Obtain destination countries or regional groupings available for travel coverage.

User Action
Customer selects destination country/region
Returns
Destination countries and regional groups
4

Get Plans & Prices

consulta_planes_grupal

Calculate and retrieve available plans with pricing based on trip duration, traveler ages, selected category, and destination.

Required Input
Trip dates, traveler ages, category, destination
Returns
Available plans with pricing details
Calculation
System calculates trip duration from dates
5

Get Benefits Optional

consulta_beneficios_adicionales
consulta_beneficios_planes_costos

Review detailed plan benefits and coverage details to help customers make informed decisions.

Purpose
Display comprehensive benefit information
Returns
Coverage details, limits, and benefits
6

Validate Coupon Optional

consulta_cupon

Apply and validate discount codes to calculate final pricing with promotional discounts.

User Action
Customer enters promo/discount code
Returns
Validated discount amount or error
7

Purchase

comprar

Execute the purchase transaction with complete traveler information and trip details to generate vouchers.

Required Data
Full traveler details, payment info, trip data
Returns
Voucher numbers and confirmation
Result
Travel assistance plan activated
8

Retrieve Voucher

consulta_voucher
consulta_beneficios_voucher

Fetch voucher confirmation details and benefits for customer records and travel documentation.

Purpose
Confirmation and documentation retrieval
Returns
Complete voucher details and coverage info

Authentication Requirements

Protocol: All endpoints require HTTPS POST with TLS 1.2 minimum

Headers:

  • PHP-AUTH-USER for legacy clients (pre-January 13, 2025)
  • EVA-AUTH-USER for clients onboarded after January 13, 2025

Response Format:

  • Success: error: false with resultado array
  • Failure: Error codes (401 unauthorized) with descriptive messages
